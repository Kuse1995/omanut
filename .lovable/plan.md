## The real problem\n\nWe already added `response_type: 'code'` to `FB.login()` — but Meta still rejects with `response_type=token is not supported in this flow`.\n\nAfter verifying against Meta docs and Stack Overflow, the cause is clear:\n\n**`FB.login()` ignores `response_type` and ALWAYS uses the implicit token flow.**\nFacebook Login for Business **requires** the authorization-code flow.\n\nSo `FB.login()` is fundamentally incompatible with Login for Business. The JS SDK only supports the legacy implicit flow. We must **stop using `FB.login()`** and instead drive Meta's OAuth dialog ourselves.\n\n## The fix\n\nReplace `FB.login()` with a **manual OAuth popup** that opens:\n\n```\nhttps://www.facebook.com/v19.0/dialog/oauth\n  ?client_id=<APP_ID>\n  &config_id=<CONFIG_ID>          ← Login for Business config\n  &response_type=code              ← honored here, unlike FB.login\n  &redirect_uri=<our callback>\n  &state=<random nonce>\n  &scope=...\n```\n\nThis is Meta's documented \
