

# Expand Gemini Embedding 2 Across the Entire Platform

## Current State

Gemini Embedding 2 is only used in **2 places**:
1. **Product catalog indexing** (`embed-catalog`) — embeds product name/description/price into `payment_products.embedding`
2. **Product search tools** (`lookup_product`, `recommend_services` in `whatsapp-messages`) — query-time embedding for semantic product search

## Expansion Areas

### 1. Media Library Semantic Search (High Impact)

**Problem:** The `send_media` tool dumps the entire media library into the system prompt as a flat list. The AI must pattern-match descriptions to find relevant media. With 50+ media items this wastes tokens and produces poor matches.

**Fix:** Add an `embedding` column to `company_media`. When `index-brand-asset` runs (Vision AI already generates description + tags), also embed that text via Gemini Embedding 2. Create a `match_media` RPC function. Replace the flat media list injection with a semantic search tool.

| Component | Change |
|-----------|--------|
| DB migration | Add `embedding vector(768)` to `company_media` |
| DB function | Create `match_media` RPC (mirrors `match_products`) |
| `index-brand-asset` | After Vision analysis, embed description+tags and store vector |
| `reindex-company-media` | Batch re-embed all media for a company |
| `whatsapp-messages` | Add `search_media` tool that does semantic search instead of injecting full list. Reduce prompt bloat. |

### 2. Knowledge Base / Document Semantic Search (High Impact)

**Problem:** All `company_documents` content is dumped into the system prompt verbatim. For companies with many documents, this causes massive token usage and context overflow.

**Fix:** Add an `embedding` column to `company_documents`. Embed parsed content at parse time. Replace full document dump with a `search_knowledge` tool that retrieves only relevant chunks.

| Component | Change |
|-----------|--------|
| DB migration | Add `embedding vector(768)` to `company_documents` |
| DB function | Create `match_documents` RPC |
| `parse-document` | After parsing, embed the content and store vector |
| `whatsapp-messages` | Add `search_knowledge` tool; remove full document dump from prompt |
| `boss-chat` | Add `search_knowledge` tool for boss queries too |

### 3. Conversation History Search (Medium Impact)

**Problem:** The AI has no way to search past conversations. When a returning customer asks "what did we discuss last time?", the AI can only load the last N messages from the current thread.

**Fix:** Embed conversation summaries (already generated by `analyze-and-followup`) and store them for semantic retrieval.

| Component | Change |
|-----------|--------|
| DB migration | Add `embedding vector(768)` to `conversations` table |
| `analyze-and-followup` | After generating summary, embed it and store |
| `whatsapp-messages` | Add `search_past_conversations` tool |

### 4. Customer Intent Embeddings (Medium Impact)

**Problem:** The `segment-customers` function uses basic keyword matching for customer categorization. No semantic understanding of customer needs.

**Fix:** Embed the first message of each conversation. Use cosine similarity to cluster customers by intent without LLM calls.

| Component | Change |
|-----------|--------|
| `segment-customers` | Embed customer messages, cluster by similarity |

### 5. Boss Chat Knowledge Access (Quick Win)

**Problem:** `boss-chat` has no access to company knowledge base or media library via semantic search — the boss can't ask "what do we know about X?" and get relevant document excerpts.

**Fix:** Expose the same `match_documents` and `match_media` RPCs as tools in `boss-chat`.

## Database Changes

```sql
-- 1. Add embedding column to company_media
ALTER TABLE public.company_media ADD COLUMN IF NOT EXISTS embedding vector(768);

-- 2. Add embedding column to company_documents  
ALTER TABLE public.company_documents ADD COLUMN IF NOT EXISTS embedding vector(768);

-- 3. Add embedding column to conversations for history search
ALTER TABLE public.conversations ADD COLUMN IF NOT EXISTS summary_embedding vector(768);

-- 4. Match media RPC
CREATE OR REPLACE FUNCTION public.match_media(
  query_embedding vector, match_company_id uuid,
  match_threshold double precision DEFAULT 0.3,
  match_count integer DEFAULT 5
) RETURNS TABLE(id uuid, description text, category text, file_path text,
  media_type text, tags text[], similarity double precision)
LANGUAGE sql STABLE SECURITY DEFINER SET search_path TO 'public' AS $$
  SELECT m.id, m.description, m.category::text, m.file_path, m.media_type,
    m.tags, 1 - (m.embedding <=> query_embedding) AS similarity
  FROM public.company_media m
  WHERE m.company_id = match_company_id AND m.embedding IS NOT NULL
    AND 1 - (m.embedding <=> query_embedding) > match_threshold
  ORDER BY m.embedding <=> query_embedding LIMIT match_count;
$$;

-- 5. Match documents RPC
CREATE OR REPLACE FUNCTION public.match_documents(
  query_embedding vector, match_company_id uuid,
  match_threshold double precision DEFAULT 0.3,
  match_count integer DEFAULT 3
) RETURNS TABLE(id uuid, filename text, parsed_content text, similarity double precision)
LANGUAGE sql STABLE SECURITY DEFINER SET search_path TO 'public' AS $$
  SELECT d.id, d.filename, d.parsed_content,
    1 - (d.embedding <=> query_embedding) AS similarity
  FROM public.company_documents d
  WHERE d.company_id = match_company_id AND d.embedding IS NOT NULL
    AND 1 - (d.embedding <=> query_embedding) > match_threshold
  ORDER BY d.embedding <=> query_embedding LIMIT match_count;
$$;

-- 6. Match conversations RPC
CREATE OR REPLACE FUNCTION public.match_conversations(
  query_embedding vector, match_company_id uuid,
  match_threshold double precision DEFAULT 0.3,
  match_count integer DEFAULT 5
) RETURNS TABLE(id uuid, phone text, customer_name text, transcript text,
  started_at timestamptz, similarity double precision)
LANGUAGE sql STABLE SECURITY DEFINER SET search_path TO 'public' AS $$
  SELECT c.id, c.phone, c.customer_name, c.transcript, c.started_at,
    1 - (c.summary_embedding <=> query_embedding) AS similarity
  FROM public.conversations c
  WHERE c.company_id = match_company_id AND c.summary_embedding IS NOT NULL
    AND 1 - (c.summary_embedding <=> query_embedding) > match_threshold
  ORDER BY c.summary_embedding <=> query_embedding LIMIT match_count;
$$;
```

## Edge Function Changes

### `index-brand-asset/index.ts`
- After Vision AI returns description+tags, call `embedText()` with the combined text
- Store the vector in `company_media.embedding`

### `parse-document/index.ts`
- After parsing content, call `embedText()` with the parsed content
- Store in `company_documents.embedding`

### `analyze-and-followup/index.ts`
- After generating conversation summary, embed it
- Store in `conversations.summary_embedding`

### `whatsapp-messages/index.ts`
- Add 3 new tools: `search_media`, `search_knowledge`, `search_past_conversations`
- Remove full media library dump from prompt (replace with tool description)
- Remove full document dump from prompt (replace with tool description)
- This dramatically reduces prompt token usage

### `boss-chat/index.ts`
- Add `search_knowledge` and `search_media` tools
- Boss can ask "what info do we have about X?" and get semantic results

### `reindex-company-media/index.ts`
- Add embedding generation to the reindex loop

## Impact

- **Token savings**: Removing full media + document dumps from prompts saves thousands of tokens per message
- **Better accuracy**: Semantic search finds relevant media/docs instead of relying on AI to scan a flat list
- **Customer history**: AI can reference past conversations intelligently
- **Boss empowerment**: Boss can query knowledge base and media semantically
- **Scales**: Works regardless of media/document library size
